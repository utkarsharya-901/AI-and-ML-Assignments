1. ObjectiveTo build and train a Reinforcement Learning agent capable of balancing a pole on a moving cart for as long as possible using the standard OpenAI Gym (or Gymnasium) CartPole environment.

2. Environment DetailsThe CartPole-v1 environment is a classic control problem in reinforcement learning.
State Space (Continuous): A 4-dimensional vector representing:

Cart Position
Cart Velocity
Pole Angle
Pole Angular Velocity

Action Space (Discrete): 2 possible actions:
0: Push cart to the left
1: Push cart to the right

Reward: The agent receives a reward of +1 for every step the pole remains upright. The environment is considered "solved" if the agent achieves an average reward of 475 over 100 consecutive episodes.

3. Methodology & Architecture (DQN)To solve this continuous-state problem, we use a Deep Q-Network (DQN).

Neural Network (Q-Network):Instead of a traditional Q-table (which fails on continuous states), we use a Multi-Layer Perceptron (MLP) built with PyTorch. It takes the 4D state vector as input and outputs the predicted Q-values (expected future rewards) for both possible actions.

Experience Replay Buffer:RL agents can become unstable if they learn from consecutive, highly correlated experiences. The Replay Buffer stores past experiences (state, action, reward, next_state, done). During training, the agent samples random batches from this buffer, breaking the correlation and stabilizing learning.

Target Network:DQN uses two identical networks. The Policy Network actively learns and chooses actions. The Target Network computes the target Q-values for the loss function and is updated less frequently. This prevents the target from constantly shifting while the policy network is trying to catch up.

Epsilon-Greedy Exploration:The agent starts by taking completely random actions (Exploration, $\epsilon = 1.0$). As training progresses, $\epsilon$ decays, and the agent begins to exploit its learned knowledge to choose the best actions.
