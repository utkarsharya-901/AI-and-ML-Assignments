1. Project Overview & Methodology:

This RAG pipeline connects a Large Language Model (LLM) to your external data. The architecture is divided into three core stages:  Ingestion (Document Loading & Chunking): Using LangChain's DirectoryLoader and RecursiveCharacterTextSplitter, it ingests your raw files and cuts them into smaller, overlapping semantic chunks (1000 characters per chunk, with 200 characters overlap to ensure no context is lost at the boundaries).  Vector Storage (Embeddings): It converts the chunks into mathematical vectors using OpenAI's text-embedding-3-small model and stores them locally on your disk in a ChromaDB vector database.  Retrieval & Conversational Generation: When a user asks a question, the system queries ChromaDB to find the top 4 most semantically similar text chunks. It feeds those specific chunks—along with the chat history—to the LLM (gpt-3.5-turbo with temperature=0 to enforce strict factual grounding) to synthesize an accurate, concise answer.  

2. Setup and Usage Instructions:

Step 1: Install Dependencies
Set up a Python virtual environment and install the packages listed in the requirements.txt file (this includes LangChain, ChromaDB, and OpenAI toolkits).

Step 2: Add your API Key & Documents

Set your OpenAI API key in your terminal environment: export OPENAI_API_KEY="your-api-key" (or set OPENAI_API_KEY="..." on Windows).

Create a folder named documents in the project root and drop any PDF or TXT files you want the chatbot to read.

Step 3: Build the Vector Database
Run the script with the ingest flag. This will process your documents and construct the ChromaDB storage

Step 4: Chat with Your Data
Run the script with the chat flag. This will launch a continuous, conversational interface right in your terminal
